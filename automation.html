# Deploy via JAMF – Details

Deployment strategies using JAMF policies and triggers.

---

## 🔄 Policy Types
- Recurring check-in
- Login/logout
- Custom triggers (e.g., `depnotify_start`)

## 🔧 Common Payloads
- Install software (.pkg or App Store)
- Run scripts (pre/post install logic)
- Reboot, inventory update, file copy

## 🔁 Custom Triggers
- Allows chaining of policies
- Can be executed from Terminal or JAMF script
- Used for DEPNotify or post-setup automation

## 🛠️ Logs & Monitoring
- Use Policy Logs in JAMF for troubleshooting
- Monitor last execution time and result
